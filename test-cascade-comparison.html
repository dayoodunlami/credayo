🔬 Cascade Engine Comparison

Basic Proximity-based vs Advanced Graph-based Simulation

🔵 Basic Engine (Current)

Approach: Proximity-based cascade propagation with simple timing

✅ Pros

  • Simple to understand and debug
  • Fast execution for real-time use
  • Already integrated and working
  • Good for basic demonstrations
  • Low memory footprint

❌ Cons

  • Lacks realistic dependency modeling
  • Random probability calculations
  • No backup power consideration
  • Limited cross-sector logic
  • Basic animation effects

🟢 Advanced Engine (New)

Approach: Graph-based BFS traversal with probability modeling

✅ Pros

  • Realistic dependency graph modeling
  • Sophisticated probability assessment
  • Backup power duration tracking
  • Advanced cross-sector rules
  • Multi-stage animation timing
  • Better economic impact modeling

❌ Cons

  • More complex to understand
  • Higher computational overhead
  • Requires dependency data setup
  • Not yet fully integrated
  • May be overkill for simple demos

Basic Engine Approach

Advanced Engine Approach

// Graph-based BFS traversal simulateCascade() { queue = [initialAsset]; while (queue.length > 0) { current = queue.shift(); // Traverse dependency graph for (dependent of current.suppliesTo) { assessment = assessFailureProbability( current, dependent, timeElapsed ); if (assessment.willFail) { queue.push(dependent); } } } } // Multi-stage animation playCascade() { // Expanding circles // Connection lines // Ripple effects // Probability-based halos }
